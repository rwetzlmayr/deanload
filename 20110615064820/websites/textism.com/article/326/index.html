Web Writing Applescripts:: 2 April 01

After stamping my feet awhile ago about the dearth of writing tools suited to producing well-formed copy for the web, I’ve written a suite of AppleScripts that approximate some of what I was yammering about: reducing the agony of coding text. The holy grail remains an application that can switch transparently between styled text and code view, but it’s a start.

With the advent of Blogger et al., and the rise of database-driven content management systems, web authors are doing a lot less page construction and a lot more pasting of text into forms. But preparing good copy for the web is cumbersome at best: formatting with tags is counterintuitive to those used to word processors. Typographic standards, consistent formatting and browser compatibility all suffer. Scripts running on web servers can take care of some formatting, but only some.

On the other hand, writing and revising copy entwined in the barbed wire of markup is murder, and previewing in a browser is too much of a snag.

So the goal here is to afford authors tools that function within a familiar word processing environment, which simplify the process of marking up text, and ensure that analphabetics such as dashes and quotation marks are properly formatted.

NB: these scripts are designed to work with Tom Bender’s excellent shareware text editor Tex-Edit Plus, though anyone with a bit of scripting chutzpah can tweak for use with any styled text editor. Er, no. They don’t work on Windows.

What the scripts do

Generic Tag Wrapper [ shift-cmd-t ]

Wraps any tag around selected text. Defaults to Span with a prompt for a Class attribute, because that’s what I use most.

Georgify [ shift-cmd-g ]

This is optional, merely a quick way to convert the writing environment to Matthew Carter’s Georgia 17pt: a solid onscreen writing setup.

Hyperlink Tag Wrapper [ shift-cmd-h ]

Wraps hyperlink tags around selected text. Most useful if you’ve copied a URL from somewhere else. Prompts for a Title attribute: leave this blank if you don’t use Title.

List Maker [ shift-cmd-L ]

Quickly formats a numeric or bulleted list. Enter values through a popup window.

Load in IE [ shift-cmd-e ]

Previews the current state of the text in Internet Explorer. Useful for checking links and formatting. Working on a way to link style sheets at the same time, but this works nicely.

Load in IE with CSS [ shift-cmd-r ] NEW

  • Creates a temporary HTML file with Latin-1 language declarations
  • Prompts for the URL of a CSS file
  • Loads the temporary page in IE and previews the text with CSS formatting
  • Remembers the last supplied CSS URL

Markup Noise Reducer [ shift-cmd-n ]

Changes the format of all tags to small, blue and Monaco. If you’ve used the Georgify script, this will leave your copy standing tall, proud and Georgified.

Preflight Cruncher [ shift-cmd-c ] UPDATED

The last step. Run this script to automatically convert dodgy characters to HTML entities, check block-level formatting, wrap text styled as bold or italic in appropriate tags, and a few other things. Updated April 22:
  • Paragraphs formatted with indentation are converted to <blockquote>
  • Entity conversions vastly expanded
  • Block formatting less error-prone

URL Grabber [ shift-cmd-u ]

A more efficient version of the Hyperlink Tag Wrapper. Useful for adding links to copy while browsing. Quietly asks Internet Explorer what the current loaded web page is, makes a hyperlink from its URL, then asks for a Title attribute, politely loading the title of the web page as a default.
