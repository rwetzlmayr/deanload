Txp bee one 3 · 27 February 03

Writing code all day is fun. Fun fun fun. Such fun. Fourteen hours a day of fun. Humbling in its funniness.

Textpattern beta 1.3 is up, largely rebuilt to deal with some security vulnerabilities, and to help it along with functioning on systems not precisely one hundred percent like the one I use. Now back to documentation and a new support forum. Won’t be trying to write the latter from scratch, this time.

One thing that had been driving me nuts the past week was the absence of clear navigation in Textpattern between the three forms text takes on its way to the web text: first as plain text, then as HTML, then as parsed in a browser. A new feature in Textpattern, illustrated here, aims to simplify the process a bit. On its composing page, text is always available in all three forms. Using tabs, a writer can glide between plain text, HTML and parsed preview before saving an article. Seems to work pretty well.

Sleep now.
